# Workflow: 배포 및 검증

## 목표
Vercel로 배포하고 PC/모바일에서 전체 기능을 검증한다.

## 입력값
- GitHub 레포지토리
- Vercel 계정

## 단계

### Step 1: GitHub 레포 생성 및 푸시
- `git init` → `git add .` → `git commit`
- GitHub에 레포 생성 후 push

### Step 2: Vercel 배포
- Vercel에서 GitHub 레포 연결
- 환경변수 설정 (.env 내용을 Vercel에 등록)
- 자동 배포 확인

### Step 3: 기능 검증 체크리스트
- [ ] 대시보드 정상 표시
- [ ] Day 목록 정상 표시
- [ ] Day 상세 페이지 진입
- [ ] 상태 변경 (미학습 → 완료 → 나중에)
- [ ] 메모 입력 및 저장
- [ ] 새로고침 후 데이터 유지
- [ ] PC 브라우저에서 정상 동작
- [ ] 모바일 브라우저에서 정상 동작
- [ ] PC에서 변경 → 모바일에서 반영 확인 (실시간 동기화)

## 결과물
- Vercel URL로 어디서든 접속 가능
- PC/모바일 모든 기능 정상 동작

## 에러 대응
- 빌드 실패 → 빌드 로그 확인, 종속성 문제 해결
- 환경변수 누락 → Vercel 대시보드에서 환경변수 추가
- Firestore 연결 실패 → Firebase 콘솔에서 도메인 허용 확인
